CmdCLD Remote Access — Setup Guide

Control your Claude sessions from any device on your network — phone, tablet, or another PC.

┌──────────────┐ Tailscale ┌──────────────┐ │ Your Phone │◄── encrypted ──►│ Home PC │ │ or Laptop │ tunnel │ (CmdCLD) │ └──────────────┘ └──────────────┘ Browser ──► http://100.x.x.x:3456 ──► Remote UI

How It Works

CmdCLD runs a lightweight web server on your home PC. You access it from any browser on your network. Tailscale creates a secure, encrypted tunnel between your devices — no port forwarding, no cloud relay, no accounts to manage.

Step 1: Set Up Tailscale on Your Home PC

1.1 — Install Tailscale

Download and install Tailscale from tailscale.com/download

Choose the installer for your OS (Windows, macOS, or Linux). Run it and sign in with your Google, Microsoft, or GitHub account.

1.2 — Verify Tailscale is running

After installing, you should see the Tailscale icon in your system tray (Windows) or menu bar (macOS). Click it and verify it says Connected.

Your PC now has a Tailscale IP address (starts with 100.x.x.x). You can find it by clicking the tray icon or running:

tailscale ip

Step 2: Set Up Tailscale on Your Remote Device

2.1 — Phone / Tablet

Install the Tailscale app from the App Store (iOS) or Google Play (Android). Sign in with the same account you used on your PC.

Toggle the connection on. Your phone is now on the same private network as your PC.

2.2 — Another PC / Laptop

Install Tailscale on the other machine (same download page). Sign in with the same account.

Both machines are now on the same Tailscale network and can reach each other directly.

No Tailscale? If both devices are on the same local network (same WiFi/LAN), you can skip Tailscale entirely. Just use the local IP address shown in CmdCLD settings.

Step 3: Enable Remote Access in CmdCLD

3.1 — Open Settings

In CmdCLD on your home PC, click the gear icon in the sidebar to open Settings.

3.2 — Enable Remote Access

Scroll down to the Remote Access section. Check the Enable Remote Access checkbox.

The default port is 3456. You can change it if needed (disable first, change, re-enable).

3.3 — Note the URLs

Once enabled, CmdCLD shows the URLs you can use to connect:

http://192.168.1.42:3456    ← local network
http://100.84.23.15:3456    ← Tailscale

Click any URL to copy it to your clipboard.

3.4 — Add Favorite Folders (recommended)

Still in Settings, add your project folders to the Favorite Folders list. These folders appear in the remote UI when you create a new session, so you don't have to type paths on your phone.

Step 4: Connect from Your Remote Device

4.1 — Open the URL

On your phone, tablet, or other PC, open a browser and navigate to the URL from step 3.3.

If using Tailscale, use the 100.x.x.x URL. If on the same LAN, use the 192.168.x.x URL.

4.2 — You're connected!

You should see the CmdCLD Remote dashboard showing all active sessions. From here you can:

  • View sessions — see which are busy or idle
  • Open a session — tap a card to see live terminal output
  • Send input — type messages to Claude (full terminal on desktop, quick actions on mobile)
  • Create sessions — start new Claude sessions in your favorite folders
  • Kill sessions — stop sessions you no longer need

Tips

Desktop vs Mobile

On a desktop browser (wider than 768px), you get a full interactive terminal with xterm.js — same experience as the CmdCLD app itself. On mobile, you get a read-only output view plus quick action buttons (yes, no, Ctrl+C, /compact, /clear) and a text input field.

Multiple Devices

Multiple devices can connect at the same time. Everyone sees the same sessions and output. Input from any device goes to the same terminal.

Reconnection

If your connection drops (phone goes to sleep, network switches), the remote UI automatically reconnects. You won't lose any session output — it's all buffered on the home PC.

Security

There's no authentication on the remote UI. Security is provided by your network:

  • Tailscale — encrypted WireGuard tunnel, only your devices can connect
  • LAN — only devices on your local network can reach it

Don't expose port 3456 to the public internet.

When You Return to the PC

Everything done remotely is fully visible on the local CmdCLD app. Sessions created remotely appear in the grid. All terminal output is preserved. It's as if you were sitting at the PC the whole time.

Troubleshooting

Can't connect?

  • Make sure Remote Access is enabled in CmdCLD Settings
  • Check that Tailscale is connected on both devices (look for the green dot)
  • Try the LAN IP if Tailscale IP doesn't work
  • Make sure the port isn't blocked by your firewall — you may need to allow it

Port already in use?

If you see an error when enabling, another app is using port 3456. Disable remote access, change the port number, and re-enable.

Firewall

Your OS may prompt you to allow CmdCLD through the firewall the first time. Click Allow. If you missed the prompt, open your firewall settings and add an inbound rule for port 3456 (or your custom port).
